# AI 开发规范指南

## 1. 任务管理规范

### 1.1 功能开发范围
- 所有功能必须在 `TODO.md` 清单范围内进行开发
- 开发顺序可以灵活调整，但必须确保功能完整性和依赖关系
- 如需开发 TODO 清单之外的功能，必须先与项目负责人确认

### 1.2 功能完成标准
- 每个功能模块完成后必须：
  1. 完成单元测试和集成测试
  2. 提供测试报告和覆盖率数据
  3. 获得项目负责人确认
  4. 更新 `TODO.md` 的完成状态

## 2. 代码规范

### 2.1 通用规范
- 所有代码必须有清晰的注释和文档
- 遵循 DRY (Don't Repeat Yourself) 原则
- 遵循 SOLID 设计原则
- 代码提交前必须通过所有 linter 检查

### 2.2 React Native 规范
- 使用 TypeScript 进行开发
- 使用函数式组件和 Hooks
- 遵循 React Native 最佳实践
- 组件必须有 PropTypes 或 TypeScript 类型定义

### 2.3 Rust 规范
- 遵循 Rust 2021 Edition 规范
- 使用 Rust 惯用模式
- 所有公共 API 必须有文档注释
- 错误处理必须使用 Result 类型

## 3. 国际化规范

### 3.1 文本处理
- 所有用户界面文本必须同时支持中英文
- 所有错误消息必须有中英文版本
- 使用 i18n 系统管理所有文本
- 避免在代码中硬编码文本

### 3.2 格式化
- 日期时间必须考虑时区
- 数字格式必须符合本地化要求
- 货币金额必须正确处理

## 4. 错误处理规范

### 4.1 错误处理基础
- 使用自定义错误类型处理业务错误
- 所有可能失败的操作必须返回 Result
- 错误信息必须包含中英文描述
- 关键错误必须记录到日志系统

### 4.2 错误分类
- 系统错误：如 IO 错误、网络错误
- 业务错误：如验证错误、权限错误
- 用户错误：如输入错误、操作错误

## 5. 异步编程规范

### 5.1 基本要求
- 所有 I/O 操作必须使用异步实现
- 使用 tokio 作为异步运行时
- 长时间运行的任务必须支持取消操作
- 必须处理并发限制和超时

### 5.2 状态管理
- 异步操作状态必须清晰可见
- 提供适当的加载和错误状态
- 实现优雅的错误恢复机制

## 6. 测试规范

### 6.1 测试覆盖
- 单元测试覆盖率必须达到 80% 以上
- 必须包含集成测试
- 关键功能必须有端到端测试
- UI 组件必须有快照测试

### 6.2 测试质量
- 测试必须独立且可重复
- 避免测试间的依赖
- 使用适当的 mock 和 stub
- 测试必须有清晰的描述

## 7. 性能规范

### 7.1 性能指标
- 应用启动时间不超过 2 秒
- UI 响应时间不超过 100ms
- 动画必须保持 60fps
- 内存使用必须合理

### 7.2 优化要求
- 实现适当的缓存策略
- 优化网络请求
- 优化资源加载
- 实现性能监控

## 8. 安全规范

### 8.1 数据安全
- 敏感数据必须加密存储
- 使用安全的加密算法
- 实现适当的权限控制
- 避免敏感信息泄露

### 8.2 通信安全
- 使用 HTTPS 进行网络通信
- 实现适当的认证机制
- 防止常见的安全漏洞
- 定期进行安全审计

## 9. 文档规范

### 9.1 代码文档
- 所有公共 API 必须有文档
- 复杂算法必须有详细说明
- 配置项必须有说明文档
- 保持文档的及时更新

### 9.2 项目文档
- 维护清晰的 README
- 提供详细的安装说明
- 记录重要的架构决策
- 提供故障排除指南
